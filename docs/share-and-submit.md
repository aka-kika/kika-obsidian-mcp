## Obsidian Forum Draft

Title:

```text
Obsidian Codex MCP: local-first MCP server for Obsidian vaults, no plugin or API key
```

Post:

```markdown
I published **Obsidian Codex MCP**, a local-first MCP server for Obsidian vaults:

https://github.com/dot-RealityTest/obsidian-codex-mcp

It lets MCP clients such as Codex, Claude Desktop, Cline, and Cursor read, search, create, update, and organize Obsidian markdown notes.

What makes it different:

- no Obsidian plugin required
- no API key required
- Obsidian does not need to be running
- works directly with local markdown files
- supports read-only mode
- supports backup-on-write before updates/deletes
- includes a daily status report script
- includes reusable agent workflow skills

The main use case is giving a local AI coding assistant safe access to a personal/project vault without setting up the Obsidian Local REST API plugin.

I would love feedback from anyone using Obsidian with Codex, Claude Desktop, or other MCP clients.
```

```markdown
I made a small open-source MCP server for Obsidian vaults:

https://github.com/dot-RealityTest/obsidian-codex-mcp

It gives MCP clients like Codex, Claude Desktop, Cline, and Cursor access to local Obsidian markdown files.

It does **not** require:

- an Obsidian plugin
- an API key
- Obsidian to be running

It supports:

- read/search notes
- list tags
- backlinks and wikilinks
- create/update notes
- read-only mode
- backup-on-write mode
- daily project status reports

I built it because I wanted a simple local filesystem path for AI agents to work with Obsidian notes without adding another plugin or HTTP API layer.
```
